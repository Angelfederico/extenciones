Auto Flow Format Guide

Use structural tokens so Auto Flow can split prompts, map references, and keep character handles stable. Descriptions can be written in any language; the separators and handles stay the same.

Token Cheat Sheet

Scene tag[V1-S1]

Optional, but useful for ordering and matching.

Frame splitimage prompt ||| video prompt

Left side creates the image. Right side animates it.

Character handle@gary_before

ASCII handle, no spaces. Use underscores for display names with spaces.

Character fields||| voice: none ||| info: notes

Optional fields after the character description.

Regular Prompts

Use one prompt per block or line. Scene tags preserve order.

[V1-S1] Establishing wide shot of a rainy neon street at midnight, reflective puddles, cinematic depth.

[V1-S2] Medium shot of a courier under a flickering sign, soaked jacket, tense expression.

Frame To Video

Format: [V#-S#] image prompt ||| [V#-S#] video prompt 
[V1-S1] Cyberpunk city skyline at night, heavy rain, blue neon reflections ||| [V1-S1] Camera tilts down from skyline to street level as rain intensifies.

[V1-S2] Courier standing near a glowing vending machine, steam vents behind him ||| [V1-S2] Slow dolly-in while jacket fabric moves in wind.

Character Reference Workflow

  1. In Character Prompts, define each reusable handle once.
  2. In Character Sources, upload source images or choose existing project/reference images for each handle.
  3. In scene prompts, reuse the exact same @handle.
  4. For existing Flow characters, map the handle to the existing Flow character instead of recreating it.
Basic: @handle character description
Advanced: @handle character description ||| voice: Flow voice name ||| info: extra notes 
@gary_before Overweight white man, plain white t-shirt, cheap athletic shorts, nervous expression. ||| voice: none

@deandre Tall tan man, chiseled body, wavy dark hair, open shirt, smug relaxed face. ||| voice: Charon

Auto Flow Packet Import

Use Paste Auto Flow Packet to paste characters, video prompts, and notes in one block. Auto Flow previews the packet before applying it into the existing Character Prompts and Video Prompts sections.

# CHARACTER PROMPTS
@test_luma Luma Test Character is a fictional young adult explorer with a teal jacket, silver backpack, short black hair, and a warm expression. Not based on any real person. ||| voice: none ||| info: curious, calm, adventurous

@test_orren Orren Test Character is a fictional young adult inventor with an amber hoodie, round glasses, curly brown hair, and a thoughtful expression. Not based on any real person. ||| voice: none ||| info: clever, nervous, precise

# VIDEO PROMPTS
[V1-S1] @test_luma stands at the edge of a glowing forest path, teal jacket catching soft blue light, cinematic fantasy realism. ||| Slow dolly-in as @test_luma looks toward the trees, curious and cautious.

[V1-S2] @test_orren kneels beside a small mechanical device on a wooden table, amber hoodie sleeves rolled up, warm workshop lighting. ||| Camera pushes in as @test_orren adjusts the device and glances toward @test_luma.

# NOTES
Style: cinematic fantasy realism, soft lighting, shallow depth of field.
Continuity: Keep @test_luma in teal jacket. Keep @test_orren in amber hoodie and glasses.
Omni: Use native Flow character chips. Do not treat plain @text as proof.
Copy this prompt into your LLM:
You are writing an Auto Flow Packet. Use # CHARACTER PROMPTS, # VIDEO PROMPTS, and # NOTES. Define every @handle before using it. Use fictional non-famous characters. Use lowercase snake_case handles. Do not invent undefined @mentions. Use ||| to split first-frame/image prompt from video motion prompt. Use @handles only where identity matters.
Packet import does not mark Omni ready. Omni/Path C stays character_setup_required until native Flow Character rows, AT_TAG_TYPE, and characterServerId proof exist. After applying a packet, use Create / Verify Native Characters. Auto Flow verifies existing mappings or creates missing native Flow Characters from the Character Prompts, then stores @handle -> characterServerId mappings only after true Character row proof exists. Source images and project grid image assets count as character_asset_created_but_not_native_character: ready for Auto Flow Classic Match mapping, not Omni/Path C submission. A native row can remain character_character_row_pending until Flow exposes it in the @ picker.

Using Characters In Scene Prompts

Use the same handle where identity matters. For long Omni scripts, bind the important speaker/identity mentions first instead of every repeated descriptive mention.

[Scene 2] Hotel hallway at night. @gary_before rushes forward and shoves @deandre against the wall.

Dialogue:
@gary_before, furious: "Who are you?"
@deandre, calm: "Back up."
Do not use spaces inside handles. Use @gary_before, not @ gary before. A display name like "Gary Before" can still map to the handle @gary_before.

Matching Modes

ModeUse ForWhat It Means
Flow Native @ Match Existing or created Flow characters Uses Google's native character chip/entity binding path.
Auto Flow Classic Match Image references for Create Image, Frame to Video, Ingredients Auto Flow's original matcher. It maps uploaded refs by handle, filename, or prompt row. It is not Omni-only.

Language-Agnostic Rules

The description text can be any language. Auto Flow only depends on the structural tokens: one prompt per line/block|||@handlevoice:info:
  • Keep @handle in ASCII with no spaces.
  • Keep ||| as the split marker.
  • Keep optional field names as voice: and info:.
  • Everything after those tokens can be natural prompt text.
